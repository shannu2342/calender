# Seed Cache Script

This script generates pre-populated cache data from the Prokerala API that can be bundled with the deployment.

## Purpose

In production (e.g., Render), the SQLite database cache is ephemeral - it gets recreated on each deployment. This means every request immediately hits the Prokerala API, which can cause:
- 502 errors when API is down
- Rate limiting issues
- Slow first requests

The solution is to pre-populate the cache with real Prokerala data and include it in the deployment.

## Usage

### 1. Credentials are already in your .env file

The script loads from `backend/.env` automatically - no need to set environment variables manually!

### 2. Configure the locations you want to cache

Edit `seedCache.js` and modify the `CONFIG` object:

```javascript
const CONFIG = {
  locations: [
    { lat: 17.8025, lng: 79.5920, name: "Karimnagar" },
    { lat: 17.3934, lng: 78.4706, name: "Hyderabad" },
    // Add more locations as needed
  ],
  startDate: "2026-01-01",
  endDate: "2026-12-31",
  languages: ["en", "te", "hi"],
  ayanamsa: 1,
};
```

### 3. Run the seed script

```bash
cd backend
node scripts/seedCache.js
```

This will:
1. Call the Prokerala API for each date/location/language combination
2. Store the results in the local SQLite database
3. Export the seed data to `backend/data/panchang_cache_seed.json`

### 4. Deploy with seed data

The seed file will be automatically loaded on server startup if:
- The database is empty
- The seed file exists

Just make sure `backend/data/panchang_cache_seed.json` is included in your deployment!

## Deployment Options

### Option A: Commit seed file (Recommended for small caches)

1. Commit `backend/data/panchang_cache_seed.json` to your repo
2. Deploy - the server will load it automatically

### Option B: Use build hook

1. Run seed script locally
2. The seed file is generated
3. Deploy triggers and includes the seed file

## Monitoring

After deployment, check server logs for:
```
[Database] Loading seed data from ...panchang_cache_seed.json...
[Database] Loaded XXX seed entries
```

This confirms the seed data is being loaded.

## Seed File Location

The seed file is stored at:
- `backend/data/panchang_cache_seed.json` (JSON format for easy inspection)
- `backend/data/panchang_cache_seed.sql` (SQL format for direct import)

Both are generated by the seed script.
